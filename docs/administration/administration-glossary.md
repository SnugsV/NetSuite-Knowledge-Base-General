# Administration Glossary

## Quick Summary

A reference of terms used in NetSuite administration. Use this glossary when you encounter an unfamiliar term in administration articles.

## A

**Administrator**
A user role with unrestricted access to all NetSuite features, records, and setup pages.

**Audit Trail**
An account-wide log of significant events — user management, role changes, configuration modifications, and login activity.

## B

**Backup Codes**
Single-use codes generated during 2FA enrollment that can be used to log in if the authenticator app is unavailable.

## C

**Center**
A navigational framework that groups related menus, dashboards, and pages around a business function (e.g., Sales, Inventory, Setup).

**Custom Field**
A user-defined field added to a standard or custom record type to capture organization-specific data.

**Custom Form**
A modified version of a standard form with different fields, layout, or behavior, often tailored to a specific role.

## D

**Default Role**
The role a user starts in when they log in. Users with multiple roles can switch to a different role during their session.

**Deactivate (User)**
Removing a user's ability to log in while preserving their transaction history and employee record.

## E

**Employee Record**
An HR record containing a person's name, department, job title, and personal information. An employee record can exist without granting NetSuite access.

**Enable Features**
The setup page where administrators turn NetSuite features on and off.

## F

**Feature Dependency**
A requirement that one feature must be enabled before another can be used (e.g., Advanced Inventory requires Inventory Management).

**Fiscal Period**
A time period used for accounting purposes (typically a month or quarter). Periods can be open, locked, or closed.

**Form**
The layout of fields, tabs, and sections used to view or edit a record.

## H

**Home Center**
The default landing page for most users, displaying KPIs, reminders, and recent records.

## I

**IP Address Rule**
A security setting that restricts NetSuite login to specific IP addresses or ranges.

**Irreversible Feature**
A feature that, once enabled, cannot be disabled (e.g., Multi-Currency, Advanced Inventory).

## L

**Least Privilege**
The security principle that users should be granted only the permissions they need to perform their work, and no more.

**Login Email**
The email address a user types to log in to NetSuite. This can differ from the user's personal email.

## O

**OneWorld**
NetSuite's enterprise edition for organizations with multiple subsidiaries, currencies, and legal entities.

## P

**Password Policy**
Rules that define minimum password requirements — length, complexity, history, maximum age, and lockout settings.

**Permission**
An individual access right that controls what a user can do with a specific feature, record type, or transaction.

**Permission Level**
The degree of access granted by a permission — typically None, View, Create, or Edit.

**Production Account**
The live NetSuite account used for daily business operations.

## R

**Release Preview**
A pre-release environment where administrators can test upcoming NetSuite features before they reach production.

**Role**
A collection of permissions and center assignments that defines what a user can access and do.

## S

**SAML (Security Assertion Markup Language)**
An XML-based protocol for exchanging authentication and authorization data between an identity provider and a service provider (NetSuite).

**Sandbox**
A copy of the production account used for testing configuration changes, customizations, and integrations.

**Session Timeout**
The length of idle time before a user is automatically logged out of NetSuite.

**Setup Center**
The NetSuite center that contains administration and configuration pages. Typically restricted to administrators.

**Single Sign-On (SSO)**
An authentication method that allows users to log in using their corporate identity provider credentials.

**System Notes**
A per-record log of field-level changes, showing who changed what, when, and the old and new values.

## T

**TOTP (Time-based One-Time Password)**
The algorithm used by 2FA authenticator apps to generate time-limited codes.

**Two-Factor Authentication (2FA)**
A security method that requires both a password and a time-based code from an authenticator app to log in.

## U

**User**
A person who can log in to NetSuite. Users are associated with employee records or external contact records.
