# Global Search

## Quick Summary

Global search is the fastest way to find any record, transaction, or saved search in NetSuite. Located in the top-right corner, it searches across all record types by name, number, or keyword.

## Difficulty

Beginner

## Estimated Time

5 minutes

## Overview

The global search field is located at the top of every NetSuite page. It works similarly to a web search engine but searches only your NetSuite account's data. It is often the quickest way to find a specific record without navigating through menus.

Global search is especially useful when:

- You know a customer name but not where it is filed
- You have a transaction number and want to open it directly
- You want to check if a record exists before creating a duplicate
- You need to quickly open a saved search by name

## How to Use Global Search

1. Press **Ctrl+G** (Windows) or **Cmd+G** (Mac) to focus the search field
2. Type a keyword (customer name, item name, document number, or partial match)
3. Review the dropdown results
4. Click the matching result to open the record

## What Global Search Can Find

| Search Target | Example |
|---|---|
| Customer name or number | "Acme Corp" or "CUST001" |
| Item name or SKU | "Steel Chair" or "CHR-STL-001" |
| Transaction number | "SO-12345" or "PO-67890" |
| Vendor name | "Supply Co" |
| Employee name | "Jane Smith" |
| Saved search name | "Low Stock Items" |
| Record type | "Customer" or "Sales Order" shows related records |

## Search Behavior

- Searches are case-insensitive
- Partial matches are supported (e.g., "Acme" finds "Acme Corporation")
- Results are grouped by record type
- The most relevant results appear at the top
- Results can include records from across the entire account (visibility depends on role permissions)

## Global Search vs. Saved Search

| Feature | Global Search | Saved Search |
|---|---|---|
| Purpose | Quick record lookup | Complex data retrieval |
| Filters | Basic keyword only | Criteria, formulas, joins |
| Results | Single record or short list | Full dataset, can be exported |
| Sharing | Personal | Can be shared by role or user |
| Persistence | Not saved | Saved and reusable |

Use global search for finding individual records. Use saved searches for lists, reports, and monitoring.

## Shortcuts and Tips

- **Ctrl+G** — Focus global search from anywhere
- **Arrow keys** — Navigate results dropdown
- **Enter** — Open selected record
- Type a record type name to see recent records of that type (e.g., typing "Customer" shows recent customers even if you do not have a specific name)
- Global search does not search the full text of records — it searches names, numbers, and key identifiers

## Common Mistakes

- **Typing long queries**: Global search works best with short keywords. Use saved searches for complex filtering.
- **Not using keyboard shortcuts**: Clicking the search field is slower than Ctrl+G.
- **Assuming search is disabled**: Global search is always available unless explicitly restricted by role permissions.
- **Expecting full-text search**: Global search does not search comments, descriptions, or memo fields.

## Best Practices

- Use Ctrl+G as your primary navigation method — it is faster than menus.
- For common records, type the first few characters of the name or number.
- When you cannot find a record via search, check whether it exists and whether your role has permission to view it.
- Combine global search with recent records for the fastest navigation.
