# Dashboards

## Quick Summary

Dashboards are customizable landing pages in NetSuite that display portlets — small information panels showing KPIs, reminders, recent records, saved searches, reports, and charts. Every center has its own dashboard, and users can personalize their dashboards to show the information most relevant to their work.

## Difficulty

Beginner

## Estimated Time

10 minutes

## Overview

A dashboard in NetSuite is the first thing a user sees when entering a center. The Home Center dashboard is typically the default landing page after login. Dashboards are built from **portlets** — resizable panels that each display a specific type of information.

Dashboards serve as your command center. A well-configured dashboard can show:

- Key performance indicators (KPIs) for your department
- Reminders for pending approvals, overdue tasks, or upcoming events
- Recent records you have worked on
- Saved searches that monitor exceptions or required actions
- Charts and reports showing trends or summaries

## Types of Portlets

### Reminder Portlets

Shows alerts for tasks requiring attention:

- Transactions pending approval (purchase orders, sales orders, credit limits)
- Opportunities closing soon
- Overdue tasks
- Upcoming events

### KPI Portlets

Displays metric-based indicators that update in real time. Common KPIs include:

- Revenue this month vs. last month
- Open sales orders
- Cash balance
- Inventory turnover
- Accounts receivable aging

KPIs can be configured to show actual values vs. targets and can include trend arrows.

### Recent Records Portlet

Shows the last 10 records the user has viewed or edited. Clicking a record opens it directly.

### Report and Saved Search Portlets

Saved searches and reports can be placed on a dashboard as portlets. This is often more useful than running reports manually. Examples:

- "Low Stock Items" saved search as a portlet on the Inventory dashboard
- "Open Orders" saved search on the Sales dashboard
- "Overdue Invoices" saved search on the accounting dashboard

### Chart Portlets

Charts can display data visually using bar, line, or pie charts. Chart data typically comes from saved searches or reports.

### Calendar Portlets

Shows events and tasks from the NetSuite calendar.

## Adding and Configuring Portlets

To customize your dashboard:

1. Navigate to the center dashboard you want to customize
2. Click **Personalize Dashboard** or **Set Up Dashboard** (location varies by theme)
3. Click **Add Portlet** to see available portlet types
4. Select the portlet type and configure its settings (e.g., which saved search to display)
5. Drag portlets to rearrange them
6. Click **Save**

See [Personalization](personalization.md) for more detail.

## KPI Dashboard

The KPI Dashboard is a specialized view that shows key metrics in a column layout. It is available from the Home Center or through Reports > KPIs. The KPI Dashboard can display:

- Revenue, gross profit, net income
- Open orders, order backlog
- Inventory value, turnover
- Employee count, sales per employee
- Custom KPIs defined by saved searches

KPIs can be compared against budgets or prior periods.

## Dashboards by Center

Each center has its own dashboard optimized for that function:

| Center | Dashboard Focus |
|---|---|
| Home | Personal KPIs, reminders, recent records |
| Sales | Sales metrics, open orders, sales by rep |
| Inventory | Stock levels, pending transfers, low stock alerts |
| CRM | Lead funnel, campaign metrics, customer activity |
| Reports | Saved search shortcuts, report history |

## Common Mistakes

- **Overloading the dashboard**: Adding too many portlets makes the dashboard slow and harder to read. Focus on the most important metrics.
- **Not using saved search portlets**: A saved search on a dashboard is often more useful than running the search manually every day.
- **Ignoring non-Home dashboards**: The Inventory dashboard is more useful than the Home dashboard for warehouse users. Switch centers to see relevant dashboards.
- **Not refreshing dashboard data**: Some portlets cache data. Use the refresh icon or refresh the page to get current values.

## Best Practices

- Start with a minimal dashboard and add portlets as you identify what you check daily.
- Use saved searches as portlets for dynamic monitoring (e.g., "Items below reorder point").
- Arrange portlets by priority — most important information at the top.
- Create separate dashboards for different purposes if your role spans multiple functions.
- Review your dashboard quarterly to remove outdated portlets and add new ones.
