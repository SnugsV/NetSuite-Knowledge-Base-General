# Saved Searches Overview

## Quick Summary

Saved searches are reusable NetSuite queries that filter records, display selected result columns, support reporting, and help users monitor business activity. They are one of the most important tools for operational visibility because they can be reused, shared, scheduled, and adapted for many departments.

## Difficulty

Beginner

## Estimated Time

5 minutes

## Overview

Saved Searches are the foundation of operational reporting in NetSuite. They allow any user — not just administrators or developers — to ask questions about their data and get immediate answers.

A Saved Search starts with a record type (transactions, items, customers, etc.), applies criteria to filter the data, and displays a customizable set of result columns. The search can be saved and run again, shared with colleagues, added to a dashboard, or emailed on a schedule.

## Key Concepts

- **Search Type** — The record type being searched, such as transaction, item, customer, vendor, or sales order
- **Criteria** — Filters that determine which records are included
- **Results** — Columns displayed in the output
- **Available Filters** — User-facing filters that allow users to refine results without editing the search
- **Audience** — Users or roles that can access the saved search
- **Public Search** — A saved search available beyond the owner, depending on permissions
- **Alerts and Emails** — Saved searches can support email notifications when configured

## Common Use Cases

- Open sales orders needing fulfillment
- Late purchase orders requiring follow-up
- Low inventory items needing reorder
- Customers with missing or incorrect data
- Items with incorrect pricing
- Transactions pending approval
- Inventory movements by location
- Orders shipped today
- Exceptions needing attention
