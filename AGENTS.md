# AI Agent Guide

## Purpose

This is the first-stop guide for AI agents working in the NetSuite Intelligence Platform repository.

## Core Rule

Treat this repository as a public-safe NetSuite reasoning system, not a private company SOP library.

## Before Editing

1. Read `README.md`.
2. Read `PROJECT_STATUS.md` and `SESSION_REPORT.md`.
3. Read `knowledge-engine/README.md`.
4. Read `knowledge-engine/MASTER_PROJECT_PROMPT.md` for content or architecture work.
5. Inspect related files before creating new ones.

## Public-Safe Boundary

Allowed: general NetSuite education, public vendor concepts, ERP reasoning models, generic troubleshooting, lifecycle explanations, retrieval guidance, and benchmark questions.

Not allowed: company SOPs, private screenshots, custom fields, saved searches, workflows, scripts, pricing, credentials, customer examples, proprietary process details, or internal prompts.

## Working Style

Make small, reviewable commits. Prefer one focused file change at a time. Do not claim success unless GitHub returns a commit SHA. If a payload is blocked or truncated, reduce scope and verify the file after committing.
