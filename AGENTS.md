# AI Agent Startup Guide

Use this file as the first stop for AI-assisted work in this repository.

## Repository Purpose

This repository is a public, general NetSuite knowledge base. It should teach broadly applicable NetSuite concepts, integration patterns, troubleshooting approaches, and AI-friendly reasoning without exposing private implementation details.

## Public-Safe Rule

Keep the public repository generic. Do not add company-specific SOPs, customer examples, custom fields, saved searches, workflows, SuiteScripts, screenshots, pricing, credentials, proprietary processes, or private implementation details.

If a detail explains how a specific organization operates, keep it in a private overlay repository or an internal knowledge source instead.

## Startup Checklist

1. Read [README.md](README.md) for repository scope and structure.
2. Read [KNOWN_GAPS.md](KNOWN_GAPS.md) for known weak or incomplete areas.
3. Read [MAINTAINER.md](MAINTAINER.md) for maintenance and public-safety expectations.
4. Read [knowledge-engine/README.md](knowledge-engine/README.md) before content or framework work.
5. Read [knowledge-engine/MASTER_PROJECT_PROMPT.md](knowledge-engine/MASTER_PROJECT_PROMPT.md) for full project operating guidance.
6. Keep changes small, reviewable, and public-safe.

## Before Publishing Changes

- Confirm the content is original and does not copy Oracle documentation.
- Confirm technical claims are sourced or clearly marked for source review.
- Confirm links and Markdown formatting are clean.
- Confirm no private or company-specific implementation details were added.
