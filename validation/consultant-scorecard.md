# Consultant Scorecard

## Purpose

Standardized scorecard for evaluating AI consultant behavior. Every benchmark evaluation produces a scored assessment across 8 dimensions.

## Scorecard

| Dimension | Weight | Score (1-10) | Weighted Score | Notes |
|---|---|---|---|---|
| **Business Understanding** | 15% |  |  | Did the AI understand the business objective before solving? |
| **Evidence Gathering** | 15% |  |  | Did the AI collect sufficient metadata and context? |
| **Reasoning** | 15% |  |  | Did the AI follow logical diagnostic steps? |
| **NetSuite Knowledge** | 15% |  |  | Did the AI correctly apply NetSuite capabilities? |
| **Solution Quality** | 15% |  |  | Is the solution technically and operationally sound? |
| **Explanation** | 10% |  |  | Did the AI teach the customer, not just answer? |
| **Confidence** | 10% |  |  | Was confidence scoring appropriate and calibrated? |
| **Escalation** | 5% |  |  | Did the AI escalate appropriately or resolve independently? |
| **Total** | 100% |  | **Final Consultant Score** | /100 |

## Scoring Guidelines

### Score 1-3: Novice
- Misses critical information
- Makes unsupported assumptions
- Skips standard diagnostic steps
- Recommends without evidence
- Cannot explain reasoning

### Score 4-6: Competent
- Gathers most required information
- Follows standard diagnostic paths
- Occasionally misses alternatives
- Provides adequate explanations
- Confidence may be slightly off

### Score 7-8: Proficient
- Collects evidence before recommending
- Follows structured reasoning
- Considers alternatives
- Explains clearly
- Appropriate confidence assessment

### Score 9-10: Expert
- Anticipates missing information before being asked
- Follows optimal diagnostic path
- Considers multiple alternatives and trade-offs
- Teaches the customer throughout the interaction
- Confidence is consistently calibrated
- Escalates only when genuinely necessary

## Target

**90+ / 100** — Production-ready consultant quality.

Below 70 requires additional training or framework improvements before the capability is considered ready.

## Scoring Dimensions Detailed

### Business Understanding (15%)

| Score | Behavior |
|---|---|
| 1-3 | Solves before understanding the problem |
| 4-6 | Asks basic questions but may miss the real objective |
| 7-8 | Confirms business objective and success criteria |
| 9-10 | Anticipates business context and downstream effects |

### Evidence Gathering (15%)

| Score | Behavior |
|---|---|
| 1-3 | Makes assumptions without verification |
| 4-6 | Gathers some metadata but misses critical items |
| 7-8 | Follows metadata checklist for the domain |
| 9-10 | Anticipates missing information, requests proactively |

### Reasoning (15%)

| Score | Behavior |
|---|---|
| 1-3 | Jumps to conclusions, skips steps |
| 4-6 | Follows general diagnostic flow but may go off-path |
| 7-8 | Follows structured diagnostic process |
| 9-10 | Follows optimal path, adapts based on findings |

### NetSuite Knowledge (15%)

| Score | Behavior |
|---|---|
| 1-3 | Incorrect capability selection |
| 4-6 | Selects correct capability but misses configuration details |
| 7-8 | Selects correct capability with appropriate configuration |
| 9-10 | Selects optimal capability, considers edge cases, explains trade-offs |

### Solution Quality (15%)

| Score | Behavior |
|---|---|
| 1-3 | Solution does not solve the problem |
| 4-6 | Solution solves the problem but has non-optimal aspects |
| 7-8 | Solution is technically and operationally sound |
| 9-10 | Solution is optimal, documented, and includes validation steps |

### Explanation (10%)

| Score | Behavior |
|---|---|
| 1-3 | Provides answer without context |
| 4-6 | Explains what but not why |
| 7-8 | Explains both what and why |
| 9-10 | Teaches the customer, anticipates follow-up questions |

### Confidence (10%)

| Score | Behavior |
|---|---|
| 1-3 | High confidence when wrong, or low confidence when correct |
| 4-6 | Generally appropriate but occasionally mis-calibrated |
| 7-8 | Confidence matches evidence quality |
| 9-10 | Consistently calibrated, transparent about limitations |

### Escalation (5%)

| Score | Behavior |
|---|---|
| 1-3 | Escalates unnecessarily or fails to escalate when needed |
| 4-6 | Generally appropriate escalation |
| 7-8 | Consistently appropriate escalation decisions |
| 9-10 | Optimal escalation — resolves everything appropriate for AI, escalates only genuine expert needs |
