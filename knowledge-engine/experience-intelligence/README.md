# Experience Intelligence Framework

## Purpose

The platform should improve from experience — just as an experienced consultant becomes more valuable over time because every customer engagement improves future engagements.

This framework teaches the platform how to evaluate completed work, capture reusable consulting knowledge, and systematically improve its own reasoning, confidence scoring, and documentation.

## Framework Documents

| Document | Purpose |
|---|---|
| [Case Template](case-template.md) | Standardized template for capturing anonymized consulting cases |
| [Resolution Patterns](resolution-patterns.md) | Recurring solution patterns extracted from completed cases |
| [Confidence Calibration](confidence-calibration.md) | Compare predicted vs. actual outcomes to improve confidence |
| [Knowledge Gap Analysis](knowledge-gap-analysis.md) | Systematic identification of missing documentation and skills |
| [Playbook Evolution](playbook-evolution.md) | How completed work feeds back into the platform's reasoning layers |

## How It Works

```
Completed case (troubleshooting, implementation, solution)
       ↓
Structured as an anonymized case in the Case Library
       ↓
Resolution Patterns extracted (recurring symptoms → fixes)
       ↓
Confidence Calibration (predicted vs. actual)
       ↓
Knowledge Gap Analysis (what was missing)
       ↓
Playbook Evolution (updates to framework layers)
       ↓
Platform improves for the next customer
```

## The Learning Cycle

```
1. Case captured → 2. Pattern extracted → 3. Confidence scored →
4. Gaps identified → 5. Playbook updated → 6. Next case benefits
```

This is not real-time learning. It is structured, human-reviewed, evidence-based evolution. Cases should be reviewed periodically (monthly or per sprint) and patterns promoted into the active framework.
