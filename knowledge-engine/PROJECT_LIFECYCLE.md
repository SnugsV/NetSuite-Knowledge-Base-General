# Project Lifecycle

## Purpose

Define the phases this project moves through from foundation through maintenance. Understanding the lifecycle helps prioritize work and set expectations.

## Phase 0: Foundation (Complete)

**Goal**: Establish structure, standards, and workflow.

- Repository structure defined
- Governance documents created
- Templates created
- GitHub workflow configured
- Project tracking files created
- Knowledge Engineering Framework built

**Exit criteria**: All governance, templates, tracking, and framework documents exist. Repository is ready for content development.

## Phase 1: Core Content (Current)

**Goal**: Build complete beginner-to-advanced content for all modules.

- Getting Started module (complete)
- Administration module
- Saved Searches module
- Inventory module (started)
- SuiteScript module (started)
- SuiteTalk module (started)
- Remaining modules (Reporting, Workflows, etc.)

**Exit criteria**: Every module in the docs/ structure has at least 3 articles covering beginner topics.

## Phase 2: Depth and Cross-Linking

**Goal**: Add depth to all modules and build strong cross-module connections.

- Multiple intermediate-level articles per module
- Advanced articles for complex topics
- Strong cross-linking between modules
- Learning paths defined for different roles
- Glossary comprehensive

**Exit criteria**: All modules have content at three difficulty levels. Cross-links form a connected knowledge graph.

## Phase 3: Community Readiness

**Goal**: Prepare the repository for external contributors and public consumption.

- LICENSE.md added
- CI/CD configured (Markdown lint checking)
- MkDocs or GitHub Pages evaluated
- Contribution guidelines proven
- Public announcement
- Community contribution guidelines tested

**Exit criteria**: Repository is public, licensed, and ready for community contributions.

## Phase 4: Maintenance

**Goal**: Keep content current with NetSuite releases and community contributions.

- Regular review cycle established
- Release notes tracked
- Deprecated content archived
- Community PRs reviewed
- Quality metrics monitored

**Exit criteria**: Content is reviewed at least quarterly and updated for each major NetSuite release.

## Relationship to Roadmap

The lifecycle phases align with the version roadmap:

| Phase | Roadmap Versions |
|---|---|
| Foundation | v0.1 |
| Core Content | v0.2 - v0.8 |
| Community Readiness | v1.0 |
| Maintenance | v1.1+ |
