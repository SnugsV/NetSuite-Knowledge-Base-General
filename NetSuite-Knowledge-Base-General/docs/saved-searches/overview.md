# Saved Searches Overview

## AI Summary

Saved searches are reusable NetSuite searches that can filter records, display selected result columns, support reporting, and help users monitor business activity. They are one of the most important tools for operational visibility because they can be reused, shared, scheduled, and adapted for many departments.

## Purpose

Saved searches help users find records, monitor exceptions, create reports, build reminders, support dashboards, and export data.

## Key Concepts

- **Search Type:** The record type being searched, such as transaction, item, customer, vendor, or sales order.
- **Criteria:** Filters that determine which records are included.
- **Results:** Columns displayed in the output.
- **Available Filters:** User-facing filters that allow users to refine results without editing the search.
- **Audience:** Users or roles that can access the saved search.
- **Public Search:** A saved search available beyond the owner, depending on permissions.
- **Alerts and Emails:** Saved searches can support email notifications when configured.

## Common Use Cases

- Open sales orders
- Late purchase orders
- Low inventory
- Customers missing data
- Items with incorrect pricing
- Transactions pending approval
- Inventory movements by location
- Orders shipped today
- Exceptions that need follow-up

## Common Mistakes

- Building criteria that unintentionally excludes needed records.
- Forgetting main line or tax line filters on transaction searches.
- Making a search public without checking the audience.
- Adding too many result columns and making the search hard to use.
- Creating duplicate searches instead of improving an existing one.

## Best Practices

- Give saved searches clear names.
- Add a short description where possible.
- Use consistent prefixes for department or purpose.
- Keep filters simple when possible.
- Test results with a known record.
- Review permissions before sharing.
- Use the audit trail when troubleshooting changes.
