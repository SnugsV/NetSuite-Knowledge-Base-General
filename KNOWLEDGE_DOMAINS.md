# Knowledge Domains

This document defines the major public knowledge domains for this repository.

Each domain should remain generic, public-safe, and useful beyond any single organization.

## Purpose

Knowledge domains help maintainers decide where new documentation belongs.

They also help AI assistants understand which platform, process, or concept should be used when answering a question.

## Domain List

| Domain | Purpose |
|---|---|
| NetSuite Core | Explains ERP concepts, records, transactions, roles, permissions, forms, reporting, automation, and integrations. |
| ERP Operations | Explains business process flows such as order to cash, procure to pay, plan to produce, and record to report. |
| Inventory | Explains items, locations, bins, lots, serials, adjustments, transfers, replenishment, and inventory valuation. |
| Manufacturing | Explains assemblies, components, work orders, builds, routings, costing, and production flow. |
| Accounting | Explains posting behavior, periods, accounts, transaction impact, reconciliation, and financial process flow. |
| Reporting | Explains saved searches, reports, dashboards, KPIs, analytics, and visibility into business activity. |
| Automation | Explains workflows, scripts, approvals, scheduled processes, and automation selection. |
| Integrations | Explains system connections, APIs, middleware, file exchange, ownership, and error handling. |
| Shipping | Explains rating, labels, carriers, fulfillment, tracking, and shipment exceptions. |
| EDI | Explains trading partner documents, acknowledgments, ASNs, invoices, mappings, and exception patterns. |
| Tax | Explains tax calculation, nexus, exemptions, taxability, address validation, and compliance concepts. |
| AI Reasoning | Explains discovery, troubleshooting, decision trees, root cause analysis, and solution design patterns. |

## Domain Guidelines

### NetSuite Core

Use this domain for foundational NetSuite concepts.

Examples:

- Records
- Transactions
- Roles
- Permissions
- Forms
- Centers
- Subsidiaries
- Locations
- Departments
- Classes

### ERP Operations

Use this domain for end-to-end business processes.

Examples:

- Order to cash
- Procure to pay
- Plan to produce
- Record to report
- Returns
- Replenishment

### Inventory

Use this domain for item and stock movement concepts.

Examples:

- Item records
- Inventory status
- Transfers

- Lot tracking

- Reorder points

### Manufacturing

- Work orders

- Routings

## Public-Safe Reminder

Do not include private company names, internal identifiers, screenshots, account-specific mappings, saved searches, custom field IDs, workflow IDs, script IDs, or internal procedures in public domain articles.

Use generic examples only.
